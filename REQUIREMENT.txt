## 项目背景

这是一个基于 Scripting app（iOS）的符合中国国情的个人闹钟管理程序。项目代码已完成 Phase 1-5 编码，TS 诊断零错误，但尚未在真机测试。以下是你需要知道的全部上下文。

## 用户需求

- 朝九晚五上班族自用
- 场景：上班/上学/放学/下班/固定周期培训班、信用卡管理闹钟、农历纪念日、节气
- 痛点：现有闹钟 app 不支持中国法定节假日调休（节假日该响不跳过、补班日该补响不补）、不好分类管理、没有多种周期（每几天/每几月/每几工作日等）、没有中国农历支持
- 典型场景：周六固定培训班，但法定节假日要自动顺延（不响）；补班日（如周日调休上班）要自动补响
- 信用卡：账单日+还款日提醒，自动计算跨月，支持宽限期和提前N天提醒

## 关键技术决策

1. 用 `Configuration.countdown` + schedule 替代 `Configuration.alarm()`（后者被 AlarmKit 拒绝）
2. UI 脚本通过 `Shell.run` 调 ios-alarm skill（UI 脚本无法直接 import skill 目录的 .ts）
3. 双层数据：Storage 存业务元数据，AlarmManager 存运行时闹钟，通过 alarmId 关联
4. weekly + holidayAware：用 AlarmManager weekly schedule + App 启动时调休预告补丁
5. monthly/yearly/lunar 模式：无法用 weekly，改为滚动创建一次性 fixed 闹钟

## Scripting app API 注意事项

- Picker 无 `Picker.Item`，用 children 放 Text 选项，`value: Observable<T>` 或 `value`+`onChanged`
- Stepper 无 value/onChanged/range，只有 `onIncrement`/`onDecrement` 回调
- TextField 无 placeholder，用 `prompt` 属性
- `foregroundStyle` 用 `"secondaryLabel"` 不是 `"secondary"`
- `fill`/`tintColor` 传字符串需 `as any` 绕过 TS 类型
- `alert()` 是全局函数，不需导入，返回 Promise
- DatePicker `value` 接受 `Observable<Date>` 或 `timestamp(number)`
- `font` 属性接受 number 或预设名，`fontWeight` 单独设
- ForEach `data` 需要 `{id: string}[]` 类型，无 id 的数据用 `.map()` 代替
- `Storage.set/get` 跨进程必须传 `{ shared: true }`
- scripting-ts 运行时对顶层 `async function` 声明有限制，必须用 IIFE `(async () => { ... })()`
- Dialog.* 取消判断用 `== null`（松散等），不要用 `=== null`
